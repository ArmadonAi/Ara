# GitHub Release Checklist

Use this checklist when publishing a new Ara release.

## Pre-release

- [ ] All tests pass: `bun test`
- [ ] Typecheck clean: `bun run typecheck`
- [ ] Build succeeds: `bun run build`
- [ ] CLI builds: `bun run build:cli`
- [ ] CHANGELOG.md updated with version and date
- [ ] RELEASE_NOTES.md updated
- [ ] All package.json versions bumped consistently
- [ ] No local absolute paths in source (`bun run doctor` checks this automatically)
- [ ] No stale .env values or real tokens committed
- [ ] .gitignore covers all generated files
- [ ] Verify `ara doctor` passes all checks with API online

## Doctor Verification Checklist

Run `ara doctor` and verify:
- [ ] API server reachable (or graceful message if offline)
- [ ] .ara directories present or noted as auto-created
- [ ] Permissions subsystem available
- [ ] Locks subsystem available
- [ ] Checkpoints subsystem available
- [ ] MCP config valid or gracefully missing
- [ ] GitHub config valid or gracefully missing
- [ ] No local path leakage (`file:///` absolute paths in source)
- [ ] Critical project files present (package.json, tsconfig.json, bun.lock, node_modules, .gitignore)

## Release Build

- [ ] Tag the release: `git tag -a v0.2.0 -m "v0.2.0"`
- [ ] Push tag: `git push origin v0.2.0`
- [ ] Create GitHub Release from tag
- [ ] Title: "Ara v0.2.0"
- [ ] Body: Summary from CHANGELOG.md
- [ ] Attach CLI binary: `apps/cli/dist/main.js` (if distributing standalone)

## Post-release

- [ ] Verify release page shows correct assets
- [ ] Clone fresh: `git clone <url> --branch v0.2.0`
- [ ] Install: `bun install`
- [ ] Test: `bun test`
- [ ] Build: `bun run build`
- [ ] CLI build: `bun run build:cli`
- [ ] Verify `ara doctor` runs without errors
- [ ] Verify `ara status` runs without errors

## Assets to include

- Source code (zip/tar.gz — auto-generated by GitHub)
- CLI bundle: `apps/cli/dist/main.js`
- Yoga WASM: `apps/cli/dist/yoga.wasm`
- No prebuilt binaries (source-only release)

## Release metadata

- Version: v0.2.0
- Status: Stable
- Target: Local developer use
- License: Proprietary/Private
