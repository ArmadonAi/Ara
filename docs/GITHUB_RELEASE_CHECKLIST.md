# GitHub Release Checklist

Use this checklist when publishing a new Ara release.

## Pre-release

- [ ] All tests pass: `bun test`
- [ ] Typecheck clean: `bun run typecheck`
- [ ] Build succeeds: `bun run build`
- [ ] CLI builds: `bun run build:cli`
- [ ] CHANGELOG.md updated with version and date
- [ ] RELEASE_NOTES.md updated
- [ ] All package.json versions bumped consistently
- [ ] No local absolute paths in source (`grep -rn "file:///"`)
- [ ] No stale .env values or real tokens committed
- [ ] .gitignore covers all generated files
- [ ] .env.example covers all required variables

## Release Build

- [ ] Tag the release: `git tag -a v0.1.0 -m "v0.1.0"`
- [ ] Push tag: `git push origin v0.1.0`
- [ ] Create GitHub Release from tag
- [ ] Title: "Ara v0.1.0"
- [ ] Body: Summary from CHANGELOG.md

## Post-release

- [ ] Verify release page shows correct assets
- [ ] Clone fresh: `git clone <url> --branch v0.1.0`
- [ ] Install: `bun install`
- [ ] Test: `bun test`
- [ ] Build: `bun run build`
- [ ] Verify `ara doctor` runs without errors

## Assets to include

- Source code (zip/tar.gz — auto-generated by GitHub)
- No prebuilt binaries in v0.1 (source-only release)

## Release metadata

- Version: v0.1.0
- Status: Release Candidate
- Target: Local developer use
- License: Proprietary/Private
