Default Deny Rules

Credential files: .env*, .ssh/**, .aws/**, .config/gcloud/**

Private keys: *.pem, *.key, id_rsa, id_ecdsa, id_ed25519

Dangerous commands: rm -rf, sudo, curl|sh, reverse shells, DROP TABLE, printenv

MCP / External Tools

Model Context Protocol integration for external tool servers.

Transports

  • stdio — Child process with JSON-RPC 2.0 over stdin/stdout
  • HTTP — JSON-RPC 2.0 POST with configurable timeout

Safety

  • 10 evaluation paths in permission mapper
  • Untrusted servers: shell denied, mutating asks, file asks
  • Full process.env never passed — strict env scoping
  • API keys redacted in audit logs

Lifecycle

stopped → starting → healthy → unhealthy/error

GitHub Integration

13 tools through the Tool Registry. Write operations require approval.

Read Tools

get_repo, list_issues, get_issue, list_pull_requests, get_pull_request, get_pull_request_files, get_pull_request_diff, list_check_runs, list_workflow_runs, get_workflow_run

Write Tools

create_issue, comment_issue, create_pull_request_review

Safety

  • Token from env var only — never stored in config
  • Redacted in all audit logs
  • Subagents cannot write
  • Repo allowlist enforcement

Subagents

Read-only delegated agents with profile-based configuration.

  • Tool allowlist with namespace wildcards (mcp.github.*)
  • Parallel scheduling with concurrency control
  • Deterministic result merging
  • AbortController-based cancellation
  • Cannot write, self-approve, or manage MCP servers

Checkpoints

  • Automatic before mutating tool execution
  • Manual via CLI: ara checkpoint create <reason>
  • Diff viewer: ara checkpoint diff <id>
  • Restore modes: code_only, conversation_only, both

Canvas Workspace

13 node types, 5 edge types. File-based persistence at .ara/canvas/workspaces/.

Node types: chat, session, task, file, artifact, memory, skill, subagent, github_issue, github_pr, mcp_tool, checkpoint, note

Skill Learning

  1. Workflow fingerprints detected from session transcripts
  2. At 3+ repetitions, a draft is auto-generated
  3. Draft saved to .ara/skill-drafts/<id>.json (secrets redacted)
  4. User reviews via CLI: ara skills draft <id>
  5. Approval writes skills/<name>/SKILL.md with versioning
