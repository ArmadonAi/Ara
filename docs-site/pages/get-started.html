Get Started

Set up Ara on your machine in under 5 minutes.

Prerequisites

  • Bun v1.3+ — JavaScript runtime and package manager
  • Node.js 22+ (for compatibility with some dependencies)
  • Git (for cloning the repository)

Installation

  1. Clone the repository 
    git clone <repo-url>
cd ara
  2. Install dependencies 
    bun install

    This installs all workspace packages and their dependencies.

  3. Configure environment 
    cp .env.example .env

    Edit .env with at least one LLM provider key:

    GEMINI_API_KEY=AIza...
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
  4. Configure GitHub (optional)

    Create .ara/github.json:

    {
  "enabled": true,
  "defaultOwner": "your-username",
  "defaultRepo": "your-repo",
  "tokenEnv": "GITHUB_TOKEN",
  "allowedRepos": ["your-username/your-repo"]
}

    Set your GitHub token: export GITHUB_TOKEN=ghp_...

  5. Start the API server 
    bun run dev:api

    Starts the Hono REST/SSE server on port 3001.

  6. Start the TUI or use CLI 
    # In another terminal:
bun run dev:cli

# Or use one-shot commands:
ara doctor
ara status
ara chat "Explain the architecture"

First Commands

ara doctor              # System diagnostics
ara status              # API server status
ara permissions         # View permission mode
ara chat "Hello"        # Chat with AI
ara github status       # GitHub integration status
ara mcp health          # MCP server health
ara canvas list         # Canvas workspaces
ara skills suggest      # Skill learning overview

Configuration

MCP Servers

Create .ara/mcp.json to register MCP tool servers:

{
  "servers": [{
    "id": "my-server",
    "name": "My MCP Server",
    "type": "stdio",
    "command": "node",
    "args": ["server.js"],
    "enabled": true
  }]
}

Lifecycle Hooks

Create .ara/settings.json to configure hooks:

{
  "hooks": [{
    "name": "git-clean",
    "type": "command",
    "events": ["SessionStart"],
    "command": "git diff-index --quiet HEAD --",
    "timeoutMs": 3000
  }]
}
