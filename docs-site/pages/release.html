What Ara Is

Ara is a local-first personal AI control plane. It gives you a secure, auditable runtime for AI agent workflows — with tools, permissions, external service integration, and full control over what the agent can do.

Recommended Use

Suitable For

  • Personal local development
  • Single-user AI workspace automation
  • MCP tool server experimentation
  • GitHub read/write with approval
  • Learning from repeated workflows

Not Yet Suitable

  • Multi-user environments
  • Public internet exposure
  • Production CI/CD pipelines
  • Untrusted third-party access

Changelog

Added

  • CLI/TUI Gateway — Commander.js with 40+ commands, React Ink TUI with 13 tabs
  • Memory & Skills — Markdown-based memory store and YAML frontmatter skill loader
  • Slash Commands — In-chat commands for all subsystems
  • Permission Engine — 5 security modes, 21 default deny rules
  • Lifecycle Hooks — 18 event types across all subsystems
  • MCP / External Tools — stdio and HTTP transports with full safety pipeline
  • GitHub Integration — 13 tools with token-redacted audit
  • File Locks — Read/write locking with deadlock detection
  • Canvas Workspace — 13 node types, 5 edge types, file-based storage
  • Skill Learning — Workflow detection, draft generation, approval/versioning
  • Checkpoints — Workspace snapshots with diff and restore
  • Subagents — Read-only profiles with parallel scheduling

Security

  • Fail-closed lock fallback — mutating operations blocked if lock module unavailable
  • File locks acquired before write_file/edit_file and mutating run_shell
  • GitHub tokens read from env var only, redacted in all logs
  • MCP audit redacts API keys, PATs, and credentials
  • Subagents cannot write, self-approve, or start/stop MCP servers

Known Limitations

SubagentsWrite-enabled parallel subagents disabled
